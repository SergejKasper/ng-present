PhoneGap

  • Open-Source-Framework
  • Apps für zehn verschiedene Plattformen entwickeln (iOS, Android, Blackberry…)
  • Bietet Schnittstellen für Geräte-Hardware wie Kamera oder GPS
Open-Source-Framework PhoneGap lassen sich Apps für sieben verschiedene mobile Betriebssysteme entwickeln: iOS, Android, Blackberry, Windows Phone, Palm WebOS, Bada und Symbian. Entwickler schreibt den gesamten Code in HTML, CSS und JavaScript. PhoneGap bietet zusätzliche Schnittstellen, um Geräte-Hardware wie Kameras oder Beschleunigungssensoren einzubinden. Der Zugriff erfolgt dabei durch JavaScript Anschließend kompiliert es alles und stellt Installationsdateien für die verschiedenen Systeme bereit. 2011 von Adobi Systems übernommen seitdem heißt es Apache Cordova

PhoneGap einrichten

  • Über die Komandozeile (NodeJS vorausgesetzt):

                        npm install -g phonegap
  • Manuell über die PhoneGap-Plattform

PhoneGap benutzen

  • Anlegen eines neuen Projektes:

phonegap create meineErsteApp --name "meineErsteApp" 
--id "de.mms.meineErsteApp"
  • Umwandlung der Web-App für die jeweilige Plattform:

                            phonegap run android

Beispiel Anwendung


<head>
<title>Gerätedaten-Beispiel</title>
<script type="text/javascript" charset="utf-8"
src="phonegap.js"> </script>
<script type="text/javascript" charset="utf-8">
    // Warten auf Cordova
    document.addEventListener("deviceready", onDeviceReady, false);
	                            
    // Cordova ist geladen.
    function onDeviceReady() {
        var element = document.getElementById('deviceProperties');
	                            
        element.innerHTML = 'Device Name: ' + device.name + '<br />' +
        'Device Cordova: ' + device.cordova + '<br />' +
        'Device Platform: ' + device.platform + '<br />' +
        'Device UUID: ' + device.uuid + '<br />' +
        'Device Version: ' + device.version + '<br />';
}
</script> 
</head>
	                              
<body>
<p id="deviceProperties">Laden der Gerätedaten...</p >
</body>

Einbinden von PhoneGap

  • Ab der Version PhoneGap 3.0:

                <script type="text/javascript" charset="utf-8"
                src="phonegap.js"> </script>
  • Vor der Version PhoneGap 3.0:

                <script type="text/javascript" charset="utf-8" 
                src="cordova-2.9.0.js"> </script>

PhoneGap Anwendung konfigurieren

  • Mit Hilfe einer config.xml Datei können: Name, Kodierung, Rechte, Icons etc. spezifiziert werden
  • Dafür muss sich config.xml im Rootverzeichnis befinden

Beipiel Config-Datei


<?xml version="1.0" encoding="UTF-8" ?>
    <widget xmlns = "http://www.w3.org/ns/widgets"
        xmlns:gap = "http://phonegap.com/ns/1.0"
        id        = "de.mms.mobileBeispiel" 
        versionCode = "10" 
        version   = "1.0.0">

    <!-- Versionscode ist nur für iOS und Android -->

    <name>PhoneGap Beispiel</name>

    <description>
        Eine Beipielandwenung für die DLC 2013. 
    </description>

    <author href="http://beispiel.de" email="ivan.demin@t-systems.de">
        Ivan Demin 
    </author>

</widget>

Access Element

  • Erlaubt den Zugriff auf Ressourcen anderer Domains
<!-- erlaubt lokale Seiten -->
<access origin="http://127.0.0.1*"/> 
<!-- individueller Zugriff -->
<access origin="https://build.phonegap.com" />
<!-- erlaubt Zugriff auf alle externe Ressourcen -->
<access origin="*" />

Ripple Emulator

Ripple Emulator

  • Google Chrome Erweiterung
  • Emuliert mobile Umgebungen
  • Erleichtert die Entwicklung und Testung von Web/Hybrid-Apps geeignet
  • PhoneGap integriert
Google Chrome Erweiterung Mit diesem Tool ist es möglich die einzelnen Bildschirmgrößen von Smartphones sowie Events wie dem drücken der Zurücktaste auf einem Androidgerät zu simulieren Nützlich um zu schauen ob das Userinterface für den Großteil von Smartphone- und Tabletgeräten geeignet ist PhoneGap ist in dem Emulator integriert kann so kann simuliert werden wie PhoneGap mit der App interagiert
